# Conceitos, Funções e Tipos de Sistemas Operacionais

A aula apresenta uma visão ampla sobre sistemas operacionais, abordando seus conceitos fundamentais, tipos existentes e aplicações práticas no mercado. Além disso, introduz ferramentas de controle de versão, com foco em Git e integração com IDEs.

## Conceito de Sistema Operacional

O sistema operacional (SO) é o software responsável por gerenciar os recursos do computador, como CPU, memória, dispositivos de entrada e saída e armazenamento. Ele atua como intermediário entre hardware e usuário, garantindo organização, segurança e eficiência.

Suas principais funções incluem:

 Gerenciamento de processos  
 Gerenciamento de memória  
 Controle de dispositivos  
 Sistema de arquivos  
 Segurança e controle de acesso  



---

## Tipos de Sistemas Operacionais

### 1. Sistemas de Grande Porte (Mainframes)

Projetados para alta capacidade de processamento e grande volume de transações simultâneas. São altamente confiáveis e seguros.

Aplicações típicas:
- Bancos  
- Grandes varejistas  
- Plataformas de e-commerce  
- Servidores web de grande escala  

Exemplos: OS/360, OS/390, Linux e variantes UNIX.

---

### 2. Sistemas Operacionais de Servidor

Focados em atender múltiplos usuários e serviços via rede. Priorizam estabilidade, escalabilidade e compartilhamento de recursos.

Exemplos:
- Linux (ampla adoção e flexibilidade)
- Windows Server (integração empresarial)

---

### 3. Sistemas de Multiprocessadores

Projetados para múltiplas CPUs ou múltiplos núcleos, exigem técnicas avançadas de escalonamento e sincronização.

Desafios:
- Balanceamento de carga  
- Sincronização (locks e semáforos)  
- Coerência de cache  

Aplicações: servidores de alto desempenho e computação científica.

---

### 4. Sistemas de Computadores Pessoais

Voltados para uso individual, com interface gráfica e suporte multimídia.

Exemplos:
- Windows  
- macOS  
- Linux  

Foco principal: usabilidade e compatibilidade de aplicações.

---

### 5. Sistemas Operacionais Portáteis

Utilizados em smartphones e tablets. Possuem forte gerenciamento de energia, segurança por permissões e integração com sensores.

Exemplos:
- Android  
- iOS  

Distribuição de aplicativos ocorre via lojas oficiais.

---

### 6. Sistemas Embarcados

Executam em dispositivos dedicados com recursos limitados, como:

- Micro-ondas  
- Smart TVs  
- Sistemas automotivos  

Exemplos mais sofisticados:
- Embedded Linux  
- QNX  
- VxWorks  

---

### 7. Sistemas de Nós Sensores

Dispositivos pequenos, com bateria limitada e comunicação sem fio. Prioridade em baixo consumo de energia.

Exemplos:
- TinyOS  
- Contiki  

Aplicações: monitoramento ambiental e agricultura de precisão.

---

### 8. Sistemas de Tempo Real

Dividem-se em:

- **Hard Real-Time**: falha no prazo pode causar desastre (ex.: controle de voo).  
- **Soft Real-Time**: atrasos são toleráveis (ex.: streaming).  

---

### 9. Sistemas de Cartões Inteligentes

Operam com recursos extremamente limitados. Foco em:

- Segurança  
- Criptografia  
- Isolamento de aplicações  

---

## Introdução ao Git

O Git é um sistema de controle de versão distribuído que permite:

- Registrar alterações no código  
- Restaurar versões anteriores  
- Sincronizar projetos com repositórios online (GitHub)  

A aula aborda:

- Instalação do Git  
- Configuração de usuário  
- Integração com VS Code  
- Criação de repositório  
- Commit, push, pull e clone  

---

## Boas Práticas de Versionamento

- Commits pequenos e frequentes  
- Mensagens claras e descritivas  
- Uso de branches  
- Testes antes de merge  

---

## Atividades Propostas

1. Configurar integração entre IDE, Git e GitHub.  
2. Criar repositório de teste, realizar commits, sincronizar e restaurar via clone.  
3. Pesquisar projetos no GitHub, clonar e analisar o conteúdo.

---

## Conclusão

A aula combina teoria e prática. Primeiro, amplia a compreensão sobre diferentes tipos de sistemas operacionais e suas aplicações. Depois, introduz o controle de versão com Git como ferramenta essencial para desenvolvimento moderno.

O objetivo final é capacitar o aluno a reconhecer tipos de sistemas operacionais, entender suas exigências de projeto e aplicar boas práticas de versionamento em ambientes reais.
